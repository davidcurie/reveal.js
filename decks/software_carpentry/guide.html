Version Control with Git

What is Git?
  • Version control is like an unlimited 'undo'
  • Version control allows many people to work in parallel
  • Works on project-wide scale
How do I Git?
  • Install Git application on machine
  • Use command lines on local machine
Getting Started
MacOS and Linux 

                            $ git config --global core.autocrlf input
Windows 

                            $ git config --global core.autocrlf true
Creating a Repository
Navigate to your file 

                            $ cd ~/Desktop
                            $ mkdir MyProject
                            $ cd MyProject
Initialize git 

                            $ git init
What did I just do?
Output 

                            Initialized empty Git repository in /Users/Dave/Desktop/MyProject/.git/
Created a hidden file .git that stores repository data
Typical Workflow
  1. Create files
  2. Take snapshot of entire directory
  3. Modify files
  4. Update snapshot
Updating snapshots
Saving a file in a directory does NOT update .git
  1. Tell .git which files to update
    • $ git add myEditedFile.txt
  2. Review all changes (optional)
    • $ git diff --staged
  3. Publish changes to repository
    • $ git commit -m "Brief summary of changes"
Other commands
  • Compare versions
  • Ignore (stop tracking) certain files
  • Share repository with others
    • Requires GitHub account (free)
